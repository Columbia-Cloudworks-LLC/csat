# Code of Conduct

## The short version

Be technical, be direct, be useful. Disagree with ideas, not people. Don't waste anyone's time with personal attacks, harassment, or chest-thumping.

## Expected behavior

- Focus on technical merit, evidence, and reproducibility
- Give feedback on code, design, and arguments — not on the person
- Assume good faith on first contact; escalate only with evidence
- Redact customer data before sharing logs, snapshots, or screenshots in any public space (issues, PRs, discussions)

## Unacceptable behavior

- Personal attacks, slurs, harassment, or sustained disruption
- Posting other people's private information
- Posting customer infrastructure data without explicit consent

## Enforcement

Project maintainers will remove or edit content that violates this code, and may bar contributors from the repo. Report concerns privately to the email listed in [`SECURITY.md`](SECURITY.md).

## Attribution

Adopted in the spirit of the [Contributor Covenant](https://www.contributor-covenant.org/), trimmed for a small, technical project.
